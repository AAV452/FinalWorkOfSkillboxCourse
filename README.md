# SkillboxSearchEngine - программа локального поискового движка по файлам.

### (создана в VSCode)
---
    Принципы работы поискового движка:
	1. В конфигурационном файле \json\config.json перед запуском приложения задаются названия файлов, по 
	   которым движок будет осуществлять поиск.
	2. Поисковый движок самостоятельно обходит все файлы и индексирует их так, чтобы потом по любому 
	   поисковому запросу находить наиболее релевантные документы.
	3. Пользователь задаёт запрос через JSON-файл \json\requests.json. Запрос — это набор слов, по которым
	   нужно найти документы.
	4. Запрос трансформируется в список слов.
	5. В индексе ищутся те документы, в которых встречаются все эти слова.
	6. Результаты поиска ранжируются, сортируются и отдаются пользователю, максимальное количество возможных 
	   документов в ответе задаётся в конфигурационном файле \json\config.json.
	7. В конце программа формирует файл \json\answers.json, в который записывает результаты поиска.

  
  Для запуска проекта:
	
1.	В папке с проектом есть архив *"nlohmann_json.rar"* (создан только для упрощения залива в репозиторий GitHub)
	  - распаковать его там же, не переименовывая. Более ничего делать не надо, все файлы настроены.
  	- сам архив удалить.
	
2. 	При открытии в VSCode, используя mingv32, собираем проект (подключение к интернету обязательно для 
	  автоматического скачивания google тестов с github.com. См: *\tests\CMakeLists.txt*).

    В коде содержатся комментарии, поясняющие логику работы программы.

4. 	После запуска автоматически проверяется наличие и корректность содержимого файла *\json\config.json*. 
	  Если с ним будут проблемы - программа выдаст соответствующие сообщения и не запустит основное меню.
  	Если меню вывелось - файл *config.json* заполнен верно.
 
5.  После десериализации списка "files" из *\json\config.json* программа автоматически проверит соответствие файлов
    для поиска из папки *\resources* со списком. Предупредит пользователя об отсутствии каких-либо файлов, а по
    каждому валидному файлу в отдельном потоке запустит индексацию: разобьёт содержимое файла на слова и
    заполнит частотный словарь (базу поисковика).
   
6.  Если возникнет ситуация, когда при работающей программе движка файлы в *\resources* были изменены или удалены, то
    перед началом поиска по запросам базу можно обновить вручную, выбрав первый пункт меню поисковика:
   
        1 - Обновить базу (заново запустить индексацию файлов из "resources\", если они были изменены)
	  
    будут выполнены действия из предыдущего пункта данной инструкции.
   
7.  Остальные пункты меню:

	Если нужно увидеть тексты валидных файлов для поиска:

        2 - Показать содержимое валидных файлов;
		
	Можно увидеть список всех уникальных слов и их количество в каждом из файлов для поиска:

        3 - Показать частотный словарь типа 'слово {индек_файла, количество_слова_в_файле}';
		
	Можно создать запрос самостоятельно и запустить поисковик по файлам только по этому запросу:

        4 - Показать релевантность 'запросу вручную';
		
	В файле *\json\requests.json* задан список поисковых запросов, его можно увидеть через:

        5 - Показать запросы из "json\requests.json"
	
	Прогнать поисковик с поисковыми запросами из *\json\requests.json* по валидным файлам из *\resources*,
	вывести результат в консоль и сохранить его в файл *\json\answers.json*:

        6 - Показать релевантность запросам из "json\requests.json" и 
		        сериализовать ответ в "json\answers.json".
